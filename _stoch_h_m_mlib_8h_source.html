38 /*!
39  @mainpage StochHMM: A Flexible C++ Hidden Markov Library and Application
40 
41  StochHMM implements HMM from simple text files. It implements traditional
42  HMM algorithms in addition to providing additional flexibility. The additional
43  flexibility is provided by allowing researchers to integrate additional data into
44  the HMM framework. For Documentation on model syntax and designing a model, see
45  Github wiki. http://www.github.com/KorfLab/StochHMM
46 
47  Here are a few of the ways that StochHMM allows the use to integrate additional
48  data sources:
49  1. Multiple Emission States
50  2. Weighting or Explicitly Defining State paths on a sequence
51  3. Linking States Emissions/Transitions to external user-defined functions
52 
53 
54  1. Multiple Emission States
55  StochHMM allows the user to provide multiple sequences. These sequences are then
56  handled by the emissions. These sequences can be REAL numbers or discrete characters/words.
57  StochHMM allows each state to have many emissions (Discrete or Continuous). Discrete emissions
58  can be independent of each other or joint distributions. The continuous emissions
59  can be considered in multiple ways. 1) They can be considered as raw probabilities which
60  will be integrated without transformation. 2) They can be considered as values to be plugged
61  into a Univariate Probability Distribution Function or Multivariate PDF (In the case of multiple
62  REAL sequences.
63 
64  Each states emissions are user-defined, so one state may have emissions from two
65  different sequences, while another may only have a single emission from a single sequence.
66 
67  2. Weighting or Explicitly Defining State paths to follow on a sequence.
68  Often, we have some prior knowledge about the sequence. If this is the case,
69  we may want to integrate that into the model, without redesigning or retraining the model (a timely endeavor).
70  StochHMM allows the user to explicitly define a State path (By name of state, or category of state).
71  In addition, StochHMM also allows the user to weight a states path (By name of State or category of state defined by user)
72  This allows the user to restrict the predicted path or weight their prior knowledge.
73 
74 
75  3. Linking States Emissions or Transitions to external user-defined functions
76  When that transition/emission is evaluated the function is called and can provide an emission.
77 
78  While this may provide one way of addressing a weakness of HMMs, which is that they do not handle long
79  range dependencies. We see it rather as a way to link together existing utilities or functions that
80  provide additional information. In this way, we can link divergent datasets or function within
81  the HMM trellis in order to arrive at a better prediction.
82 
83 
84 
85  Traditional Decoding Algorithms:
86  - Forward
87  - Backward
88  - Viterbi
89  - Posterior
90  - N-best Viterbi
91

93  Additional Stochastic Sampling Decoding Algorithms:
94  - Stochastic Forward

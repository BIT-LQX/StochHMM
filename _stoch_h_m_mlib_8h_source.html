38 /*!
39  @mainpage StochHMM: A Flexible C++ Hidden Markov Model Library and Application
40 
41  StochHMM is a free, open source C++ Library and application that implements HMM
42  from simple text files. It implements traditional HMM algorithms in addition to
43  providing additional flexibility. The additional flexibility is provided by
44  allowing researchers to integrate additional data into the HMM framework.
45 
46  For documentation on model syntax and designing a model, see Github wiki.
47 
48  ###http://www.github.com/KorfLab/StochHMM/wiki
49 
50  ##Integrating Data
51  Here are a few of the ways that StochHMM allows the users to integrate additional
52  data sources:
53  1. Multiple Emission States
54  2. Weighting or Explicitly Defining State paths on a sequence
55  3. Linking States Emissions/Transitions to external user-defined functions
56 
57 
58  ##Multiple Emission States
59 
60  StochHMM allows the user to provide multiple sequences. These sequences are then
61  handled by the emissions. These sequences can be REAL numbers or discrete characters/words.
62  StochHMM allows each state to have many emissions (Discrete or Continuous). Discrete emissions
63  can be independent of each other or joint distributions. The continuous emissions
64  can be considered in multiple ways. 1) They can be considered as raw probabilities which
65  will be integrated without transformation. 2) They can be considered as values to be plugged
66  into a Univariate Probability Distribution Function or Multivariate PDF (In the case of multiple
67  REAL sequences.
68 
69  Each states emissions are user-defined, so one state may have emissions from two
70  different sequences, while another may only have a single emission from a single sequence.
71 
72  ##Weighting or Explicitly Defining State paths to follow on a sequence.
73 
74  Often, we have some prior knowledge about the sequence. If this is the case,
75  we may want to integrate that into the model, without redesigning or retraining the model (a timely endeavor).
76  StochHMM allows the user to explicitly define a State path (By name of state, or category of state).
77  In addition, StochHMM also allows the user to weight a states path (By name of State or category of state defined by user)
78  This allows the user to restrict the predicted path or weight their prior knowledge.
79 
80 
81  ##Linking States Emissions or Transitions to external user-defined functions
82 
83  When that transition/emission is evaluated the function is called and can provide an emission. While
84  this may provide one way of addressing a weakness of HMMs, which is that they do not handle long
85  range dependencies. We see it rather as a way to link together existing utilities or functions that
86  provide additional information to the decoding algorithms. In this way, we can link divergent
87  datasets or functions within the HMM trellis in order to arrive at a better prediction.
88

90  ##Brief list of features implemented in StochHMM:
91  - General settings within Hidden Markov Models
92  1. User-defined HMM model via simple human readable text file
93  2. User-defined Alphabet
94  3. User-defined Ambiguous Characters
95  - States
96  1. Emissions
97  - Multiple emission states (Discrete / Continuous)
98  - Independent (Single or Multiple Discrete)
99  - Joint Distribution (Multiple Discrete)
100  - Univariate PDF (Single Sequence - Continuous)
101  - Multivariate PDF (Multiple Sequence - Continuous)
102  - Linkable to user-defined function
103  2. Transitions
104  - Standard Transitions
105  - Lexical Transitions (Single or multiple emission)
106  - Explicit Duration Transitions
107  - Linkable to user-defined functions
108  - Decoding
109  1. Traditional Decoding Algorithms
110  - Forward/Backward/Posterior
111  - Viterbi
112  - N-best Viterbi
113  2. Stochastic Sampling Decoding Algorithms
114  - Stochastic Forward
115  - Stochastic Viterbi
116  - Stochastic Posterior
117  - Decoding Traceback Path output formats
118  - State Path Index
119  - State Path Label
120  - GFF
121  - Hit Table (Stochastic Algorithms)
